Phase 1 — Planning

Problem Statement

Why does MediBridge need to exist?

🔍 The Core Problem

  • Patients lack real-time, personalized health guidance
  • Booking appointments is fragmented and time-consuming
  • No unified platform for diet, wellness, and AI care
  • Doctors lack digital tools to manage patient interactions
  • Mental health tracking is ignored in most HMS platforms

💡 Why MediBridge is the Solution

  • One unified platform for patients & doctors
  • AI health assistant (Gemini 2.0) available 24/7
  • Proactive health insights before disease onset
  • Integrated appointment booking via Razorpay
  • Mental wellness tracking with guided journaling

📊 Market Gap

India has <1 doctor per 1,000 patients. AI-assisted triage can reduce care burden by up to 40%.

🎯 Target Scale

10,000+ users in Tier 1 & Tier 2 Indian cities within Year 1 of launch.

⚡ Urgency

Post-COVID, digital health adoption grew 300%. Time-to-market is critical for first-mover advantage.

System Requirements

  • Personalised diet plan generation

Phase 3 — System Design

Database Design

Supabase PostgreSQL — 5 core tables

👤 users

  • id (UUID, PK)
  • email, full_name
  • avatar_url
  • plan (free / pro)
  • credits_remaining
  • created_at

🧑‍⚕️ doctors

  • id (UUID, PK)
  • user_id (FK → users)
  • name, specialty
  • qualification, bio
  • availability_slots (JSON)
  • consultation_fee

📅 appointments

  • id (UUID, PK)
  • patient_id (FK → users)
  • doctor_id (FK → doctors)
  • appointment_date
  • status (pending / confirmed)
  • payment_id (FK)

💳 payments

  • id (UUID, PK)
  • user_id (FK → users)
  • razorpay_payment_id
  • razorpay_order_id
  • amount, currency
  • status (success / failed)
  • created_at

📝 wellness_entries

  • id (UUID, PK)
  • user_id (FK → users)
  • mood_score (1–10)
  • journal_text
  • ai_insight (Gemini output)
  • entry_date

✅ Why Agile?

  • Requirements evolved throughout build
  • Rapid user feedback incorporated mid-sprint
  • Small 3-person team suits iterative cycles
  • Continuous deployment via Vercel CI/CD
  • Features released incrementally (MVP → Full)

📋 Agile Practices

  • 2-week sprint cycles
  • Daily standup (Discord / WhatsApp)
  • GitHub for version control & PR reviews
  • Backlog tracked via GitHub Issues
  • Sprint retrospective after each cycle

❌ Why Not Waterfall?

  • Requirements were not known fully upfront
  • AI components required constant tuning
  • Payment integration needed iterative testing
  • No room for late-stage design changes

Phase 6 — Implementation

Deployment & Launch

How MediBridge is deployed, trained, and migrated
🚀

Deployment Strategy

  • Frontend: Vercel auto-deploy from GitHub main
  • Backend: Render.com Node.js Express API
  • Database: Supabase managed PostgreSQL
  • Storage: Supabase Storage (CDN-backed)
  • CI/CD: GitHub push → Vercel webhook
🎓

Training Requirements

  • Doctor onboarding: in-app guided tour (5 min)
  • Patient onboarding: health profile wizard
  • Admin docs: internal Notion wiki
  • Video walkthroughs for core features
  • AI chatbot as first-line user support
🔄

Data Migration Plan

  • Greenfield project — no legacy system
  • Supabase migrations via SQL scripts
  • Seed files for initial doctor profiles
  • Env variables managed via Vercel Dashboard
  • Rollback: Git revert + Supabase snapshots

Creative Component

What Makes MediBridge Unique?

Creative differentiators that set MediBridge apart
🤖

Gemini 2.0 AI Integration

Not just a chatbot — MediBridge uses Google Gemini 2.0 Flash for context-aware medical conversations, AI-generated diet plans, mood analysis, PDF summarization, and personalised wellness insights. Chat history persists across sessions via Supabase.

🎨

Premium Design System

Glassmorphism UI, Framer Motion animations, custom cursor design, React Confetti on achievements, and full dark/light mode. Built with accessibility-first Radix UI — keyboard navigable throughout.

💳

Full Razorpay Payment Suite

Live payment gateway with order creation, HMAC signature verification, webhook processing for recurring mandates, and replay-attack protection. Supports both one-off appointments and monthly subscription billing.

🔒

Enterprise-Grade Security

Supabase Row-Level Security on every table, JWT token validation, Google OAuth, QR code scanner for quick login, and a network-resilient Supabase proxy fallback for ISP-blocked networks (e.g., Jio).
